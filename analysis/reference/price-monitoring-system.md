# Price Monitoring System

## Overview

The Hextech Analytics price monitoring system tracks market price changes for Riftbound cards and generates actionable alerts for traders and collectors.

## Architecture

```
┌─────────────────────┐
│  fetch-prices.mjs   │  Fetches current prices from TCGPlayer
│  (run: daily cron)  │  
└──────────┬──────────┘
           │
           ├─ BEFORE FETCH: Rotate prices.json → prices-previous.json
           └─ AFTER FETCH:  Save new prices → prices.json
           
┌─────────────────────┐
│ price-alerts.mjs    │  Compares current vs previous prices
│ (run: after fetch)  │  
└──────────┬──────────┘
           │
           └─ OUTPUT: price-alerts.json (market movers summary)

┌─────────────────────┐
│  PriceMovers UI     │  React component displays alerts
│  (Next.js page)     │  
└─────────────────────┘
```

## Data Files

### `public/data/prices.json`
Current price snapshot. Generated by `fetch-prices.mjs`.

**Structure:**
```json
{
  "lastUpdated": "2024-02-10T12:00:00Z",
  "cards": {
    "123456": {
      "name": "Arcane Bolt",
      "set": "foundations",
      "rarity": "rare",
      "normal": {
        "marketPrice": 18.15,
        "listedMedianPrice": 17.99,
        "buylistMarketPrice": 12.50
      },
      "foil": { ... }
    }
  },
  "stats": { ... }
}
```

### `public/data/prices-previous.json`
Previous price snapshot. Automatically rotated by `fetch-prices.mjs` before each fetch.

**Structure:** Same as `prices.json`

### `public/data/price-alerts.json`
Market movers summary. Generated by `price-alerts.mjs`.

**Structure:**
```json
{
  "generated": "2024-02-10T12:05:00Z",
  "comparisonPeriod": {
    "current": "2024-02-10T12:00:00Z",
    "previous": "2024-02-09T12:00:00Z"
  },
  "alerts": {
    "topGainers": [...],      // Top 10 price increases
    "topLosers": [...],       // Top 10 price decreases
    "extremeMovers": [...],   // ≥50% change
    "majorMovers": [...]      // ≥25% change
  },
  "stats": {
    "totalCardsTracked": 500,
    "cardsWithChanges": 120,
    "notableChanges": 45,
    "avgPercentChange": 2.3
  }
}
```

## Scripts

### `scripts/fetch-prices.mjs`

Fetches current card prices from TCGPlayer API.

**Usage:**
```bash
# Full fetch (all cards)
node scripts/fetch-prices.mjs

# Test mode (first 20 cards only)
TEST_MODE=true node scripts/fetch-prices.mjs
```

**What it does:**
1. **Rotates prices** (copies `prices.json` → `prices-previous.json`)
2. Fetches all cards from Riftcodex API
3. Filters cards with TCGPlayer IDs
4. Fetches price data from TCGPlayer (with retry logic)
5. Generates `prices.json` with market data + stats

**Rate Limiting:**
- Exponential backoff: 1s → 30s max
- Max retries: 5 per card
- Batch size: 10 concurrent requests
- 500ms delay between batches

**Output:**
- `public/data/prices.json` - Full price dataset
- `price-fetch-failures.json` - Log of failed fetches (if any)

### `scripts/price-alerts.mjs`

Compares current vs previous prices and identifies market movers.

**Usage:**
```bash
node scripts/price-alerts.mjs
```

**What it does:**
1. Reads `prices.json` (current)
2. Reads `prices-previous.json` (baseline)
3. Calculates percentage change for all cards
4. Filters for notable changes (≥10%)
5. Generates alert summary
6. Saves to `price-alerts.json`

**Alert Thresholds:**
- **Notable:** ≥10% change (up or down)
- **Major:** ≥25% change
- **Extreme:** ≥50% change

**Output:**
- `public/data/price-alerts.json` - Market movers summary

**Note:** First run will create an empty alert file (no baseline available).

## Cron Setup

For automated daily price monitoring:

```bash
# Edit crontab
crontab -e

# Add these lines (runs at 6 AM daily):
0 6 * * * cd /path/to/hextech-analytics && node scripts/fetch-prices.mjs >> logs/price-fetch.log 2>&1
5 6 * * * cd /path/to/hextech-analytics && node scripts/price-alerts.mjs >> logs/price-alerts.log 2>&1
```

**Important:** 
- Fetch prices FIRST (rotates old data)
- Run alerts 5 minutes AFTER fetch completes
- Ensure `logs/` directory exists

## Manual Workflow

For manual price monitoring:

```bash
cd /path/to/hextech-analytics

# Step 1: Fetch latest prices (includes rotation)
node scripts/fetch-prices.mjs

# Step 2: Generate alerts
node scripts/price-alerts.mjs

# Step 3: View results
cat public/data/price-alerts.json | jq '.alerts.topGainers[:5]'
```

## Troubleshooting

### No previous price data
**Symptom:** `price-alerts.mjs` reports "No previous price data found"

**Cause:** This is normal on first run. The system needs two price snapshots to compare.

**Solution:** Run `fetch-prices.mjs` twice (24 hours apart) to establish baseline.

### Rate limiting (403 errors)
**Symptom:** TCGPlayer API returns 403 errors during fetch

**Cause:** Too many requests in short time window

**Solution:** Script has built-in exponential backoff. Wait for retries to complete. If persistent, check `price-fetch-failures.json` for details.

### Missing cards in alerts
**Symptom:** Expected cards don't appear in movers list

**Check:**
1. Does the card have a TCGPlayer ID? (Check `prices.json`)
2. Did the card exist in previous snapshot? (Check `prices-previous.json`)
3. Is the price change ≥10%? (Only notable changes are included)

### Stale data
**Symptom:** `lastUpdated` timestamp is old

**Cause:** Cron job may not be running

**Solution:** 
```bash
# Check cron status
crontab -l

# Check logs
tail -f logs/price-fetch.log
tail -f logs/price-alerts.log
```

## Performance Notes

- **Fetch time:** ~5-10 minutes for 655 cards (with retry backoff)
- **Alert time:** <5 seconds (local JSON comparison)
- **Data size:** ~60KB per price snapshot
- **Memory:** <50MB during fetch process

## Security

- No API keys stored in scripts (TCGPlayer API is public/readonly)
- Output files are static JSON (safe to deploy publicly)
- No user data or sensitive information in price files
